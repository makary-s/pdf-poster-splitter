# PDF Poster Splitter — Спецификация

## Обзор

Десктопный инструмент для разрезания крупноформатных PDF-файлов (постеры, выкройки) на тайлы меньшего формата для печати на обычном принтере. Тайлы включают поля склейки с перекрытием, которые позволяют совместить соседние листы и склеить их, восстановив оригинал. Каждый тайл подписан координатами сетки; линии отреза обозначены пунктиром.

Дополнительно: на оригинал можно нанести водяной знак из логотипа, а перед каждым выходным файлом вставить титульную страницу.

Инструмент рекурсивно обходит исходную папку, обрабатывает все найденные PDF-файлы и записывает результаты в папку назначения, сохраняя оригинальную структуру поддиректорий.

---

## Интерфейс

### Версия

Отображена в верхенм правом углу.
Берется из файла build_version.
При сборке просят явно указать версию и она сохраняется в этот файл.

### Форма с настройками

#### Группа "Папки"

- **Из папки** — `inputPath` — путь (пикер)
  - Директория с исходными файлами.
  - Подпись: "PDF-плакаты из этой папки будут разбиты на листы"
- **В папку** — `outputPath` — путь (пикер)
  - Директория для сохранения результатов.
  - Подпись: "Сюда сохранятся готовые файлы, структура подпапок сохраняется"

#### Группа "Нарезка"

- **Формат бумаги** — `tileSize` — селект стандартных форматов (A4 и т.д.)
  - Размер тайла в мм.
  - Подпись: "На листы этого формата будет нарезан каждый файл"
- **Отступ для склейки (мм)** — `glueLine.margin?` — число
  - Ширина поля перекрытия в мм.
  - Подпись: "Перекрытие между листами для точного совмещения"
- **Стратегия склейки** — `glueLine.strategy` — селект
  - Подпись: "Trailing — только справа и снизу; All — все внутренние стороны"

#### Группа "Вотермарки"

- **Логотип** — `logo?` — путь до PNG (пикер)
  - Изображение для водяного знака.
  - Подпись: "PNG-картинка, накладывается на каждый плакат"
- **Ширина логотипа (мм)** — `logoWidth?` — число
  - Ширина одного экземпляра логотипа в сетке водяного знака; высота вычисляется автоматически с сохранением пропорций PNG.
  - По умолчанию 300 мм.
  - Подпись: "Высота вычисляется пропорционально"
- **Прозрачность логотипа (%)** — `logoOpacity?` — число (0–100 %)
  - Прозрачность каждого экземпляра логотипа на сетке водяного знака.
  - По умолчанию 30 %.
  - Подпись: "0 — полностью прозрачный, 100 — полностью непрозрачный"

#### Группа "Титульный лист"

- **Файл** — `titlePage?` — путь до PDF (пикер)
  - PDF-файл, добавляется первой страницей к каждому результату.
  - Подпись: "PDF-файл, добавляется первой страницей к каждому результату"

### Кнопка "Сбросить к настройкам по-умолчанию"

Сбрасывает приложение к дефолтам

#### Поведение

Любое валидное изменение поля автоматически сохраняется и восстанавливается при следующем запуске приложения.

Если поле заполнено, но содержит невалидное значение — оно подсвечивается красным с сообщением об ошибке.

При сжатии страницы по вертикали - появляется прокрутка.
Сжать можно минимум до выстоты примено 3 полей (фиксированное число).
Начальная высота - в конфиге.

### Экшн панель

#### Кнопка «Начать»

Запускает `@Flow:ProcessFolder`.

Задизейблена, если не заполнены все обязательные поля (`inputPath`, `outputPath`, `tileSize`).

#### Кнопка «Отменить»

Заменяет кнопку «Начать» во время выполнения `@Flow:ProcessFolder`. Останавливает обработку.

#### Индикаторы прогресса

Два независимых индикатора:

- **Файлы**: текущий файл из общего числа найденных PDF.
- **Тайлы**: текущий тайл из общего числа тайлов активного файла.

#### Поведение

Зафиксировано внизу экрана. Всегда видно. Не скроллится по вертикали.

## Конфигурация

Константы, задаваемые в конфиге приложения (не через UI):

- `glueLine.color` — цвет линии склейки
- `glueLine.lineStyle` — стиль линии склейки (пунктир и т.д.)
- `watermarkAngle` — угол наклона сетки водяного знака

## Flows

### @Flow:ProcessFolder

**Input:**

- `inputPath` — существующий путь к директории
- `outputPath` — существующий путь к директории
- `ctx`:
  - `tileSize: { width: mm, height: mm, name: string }`
  - `logo?`
  - `logoWidth?: mm`
  - `logoOpacity?: %`
  - `titlePage?`
  - `glueLine?` — если не передан, используется значение по умолчанию

**Шаги:**

1. Рекурсивно сканирует `inputPath`, формируя список всех PDF-файлов. Для каждого фиксирует: `pdfFileName` и `targetPath` (= `outputPath` + относительный путь внутри `inputPath`).

- Если в `outputPath` уже есть планируемые файлы (хотя бы один), вывести предупреждение о том что они будут перезаписаны. Если пользователь не согласился - завершить процесс.

2. Для каждого найденного файла запускает `@Flow:ProcessFile`.

### @Flow:ProcessFile

**Input:**

- `pdfFileName`
- `targetPath`
- `ctx`

**Шаги:**

1. `currentFile` ← первая страница `pdfFileName`.
2. Если задан `logo`:
   - `currentFile` ← `@Flow:ApplyWatermark { pdfFile: currentFile, logoWidth: ctx.logoWidth ?? 50, logoOpacity: ctx.logoOpacity ?? 30, ctx }`
   - Сохраняет `currentFile` в `targetPath` под оригинальным именем.
3. `tiledPDF` ← `@Flow:SplitIntoTiles { pdfFile: currentFile, ctx }`
4. Если задан `titlePage`: добавляет страницы `titlePage` в начало `tiledPDF`.
5. Сохраняет `tiledPDF` в `targetPath` с именем `<original_name>_<ctx.tileSize.name>.pdf`.

### @Flow:SplitIntoTiles

**Input:**

- `pdfFile`
- `tileSize: { width: mm, height: mm }`
- `glueLine: { color, margin: mm, lineStyle }`

Делит первую страницу входного PDF на тайлы и возвращает их как страницы одного выходного PDF.

**Алгоритм разбивки (по горизонтали; аналогично по вертикали):**

1. Начало — левый край страницы.
2. Отмечаем правую границу текущего тайла на расстоянии `tileSize.width`.
3. Если граница достигла или вышла за правый край страницы — тайл последний в строке, `hasRight = false`. Сравнение с допуском ≤ 0.5 pt (~0.18 мм), чтобы различия в округлении размеров страницы между PDF-генераторами не порождали лишних тайлов.
4. Если граница не достигла края — `hasRight = true`. Следующий тайл начинается с текущей позиции минус `glueLine.margin` (перекрытие).
5. Повторяем до конца страницы.

По вертикали `hasBottom` определяется по той же логике.

**Стратегия нанесения glueLine:**

glueLine — красная линия (цвет и стиль из конфига: `glueLine.color`, `glueLine.lineStyle`). Стратегия нанесения задаётся через `glueLine.strategy`:

**`trailing`** (по умолчанию): линии только со стороны следующего тайла:
- Справа — если `hasRight` (тайл не последний в строке)
- Снизу — если `hasBottom` (тайл не последний в колонке)

**`all`**: линии со всех внутренних сторон (где есть сосед):
- Справа — если `hasRight`
- Снизу — если `hasBottom`
- Слева — если `hasLeft` (тайл не первый в строке)
- Сверху — если `hasTop` (тайл не первый в колонке)

На внешнем периметре постера линии не наносятся ни в одной стратегии.

Линия рисуется вдоль внутреннего края поля перекрытия — на расстоянии `glueLine.margin` от внешней границы тайла, параллельно ей.

**Пересечение линий:** каждая линия обрывается в точке пересечения с перпендикулярной. Угловые области (двойное перекрытие) остаются без линий:

- Вертикальные (левая/правая): от `(hasTop ? margin : 0)` до `(hasBottom ? H−margin : H)`
- Горизонтальные (верхняя/нижняя): от `(hasLeft ? margin : 0)` до `(hasRight ? W−margin : W)`

**Нумерация тайлов** (формат `col.row`, например `2.3`) всегда располагается в правом нижнем углу тайла:

- Тайл имеет `glueLine` только справа → текст повёрнут на 90° (читается снизу вверх), прижат к нижнему правому углу внутри правого поля перекрытия.
- Тайл имеет `glueLine` снизу или не имеет `glueLine` вообще → текст горизонтальный, прижат к нижнему правому углу.

**Рендеринг:**

Содержимое исходного PDF хранится в выходном файле один раз и используется всеми тайлами по ссылке (XObject). «Обрезка» достигается позиционированием постера внутри границы страницы каждого тайла — содержимое физически не удаляется.

### @Flow:ApplyWatermark

**Input:**

- `pdfFile`
- `logo` — PNG-изображение
- `logoWidth: mm` — ширина одного экземпляра логотипа; высота выводится из пропорций PNG
- `logoOpacity: %` — прозрачность каждого экземпляра логотипа (0–100 %)
- `watermarkAngle` — из конфига

Создаёт сетку из копий логотипа, покрывающую всю поверхность постера.

- Сетка повёрнута на `watermarkAngle`.
- Нечётные строки сдвинуты горизонтально на половину ширины логотипа плюс зазор (брик-паттерн).
- Каждый экземпляр логотипа — ссылка (XObject), а не копия.
- Оверлей рендерится как отдельный PDF-слой, накладываемый поверх страницы постера.

**Алгоритм полного покрытия:**

Чтобы повёрнутая сетка гарантированно покрывала все углы страницы, вычисляется AABB (axis-aligned bounding box) страницы в системе координат повёрнутой сетки. Его размеры:

```
r1 = watermarkAngle (рад)
r2 = 90° - watermarkAngle (рад)

gridH = pageW * cos(r2) + pageH * sin(r2)
gridW = pageW * sin(r2) + pageH * cos(r2)
```

Якорная точка (левый верхний угол сетки в координатах страницы) — угол повёрнутого AABB, из которого строки идут горизонтально:

```
anchorX =  h1 * sin(r1)        где h1 = pageW * cos(r2)
anchorY =  pageH + h1 * cos(r1)   (в y-down координатах рендерера)
```

Сетка заполняется от якоря вверх (строки `0..rowNum` вычитаются из `anchorY`) и вправо (`0..colNum`). После построения всей сетки к ней применяется поворот на `watermarkAngle` вокруг якоря.

Количество строк и столбцов:

```
logoSpan = logoW / 2 + gap
colNum   = floor((gridW + logoSpan * 2) / logoW)
rowNum   = floor(gridH / logoH)
```
